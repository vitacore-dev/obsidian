## Назначение

Карта конфигурации помогает быстро понять, где задаются параметры SMEV Viewer и какой источник имеет приоритет.

## Файлы конфигурации

- `.env` — переменные окружения процесса (`SMEV_MONITOR_PORT`, `SMEV_MONITOR_DATA`, `SMEV_WATCH_DIRS`, `SMEV_SCAN_INTERVAL_MS`, `SMEV_ADAPTER_CONFIG`).
- `data/monitor-config.json` — конфигурация каталогов мониторинга и параметров сканирования.
- `data/adapter-config.json` — параметры подключения/адаптера СМЭВ (файл не хранится в репозитории, создается из примера).

## Приоритет источников (практика)

1. Переменные окружения (`.env` / systemd Environment).
2. `monitor-config.json`.
3. Значения по умолчанию в приложении.

Если значение задано в `env`, оно обычно переопределяет настройки в JSON.

## Обязательный минимум перед запуском

1. Проверить, что существует `adapter-config.json`.
2. Проверить список директорий в `monitor-config.json` или `SMEV_WATCH_DIRS`.
3. Проверить путь к данным (`SMEV_MONITOR_DATA`) и права записи.
4. Проверить порт API (`SMEV_MONITOR_PORT`) и доступность с хоста.

## Частые ошибки конфигурации

- Неверный путь к каталогу XML -> пустой журнал.
- Отсутствует `adapter-config.json` -> ошибки инициализации адаптера.
- Недостаточно прав на директории -> файлы не читаются/журнал не обновляется.
- Конфликт порта -> backend не стартует.
