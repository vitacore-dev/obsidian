              DECIMAL в Microsoft SQL Server

ОПРЕДЕЛЕНИЕ
═══════════
DECIMAL (и его синоним NUMERIC) — тип данных для хранения точных 
числовых значений с фиксированной точностью и масштабом.

СИНТАКСИС
══════════
DECIMAL[(p[, s])]
NUMERIC[(p[, s])]

где:
• p (precision) — общее количество цифр (слева + справа от запятой)
                  максимум: 38
• s (scale)     — количество цифр после запятой, по умолчанию: 0

ПРИМЕРЫ
════════
DECIMAL(5,2)  → всего 5 цифр, из них 2 после запятой
              Диапазон: от -999.99 до 999.99

DECIMAL(7,3)  → всего 7 цифр, 3 после запятой
              Диапазон: от -9999.999 до 9999.999

DECIMAL(10,0) → всего 10 цифр, 0 после запятой (как целое число)
              Диапазон: от -9 999 999 999 до 9 999 999 999

DECIMAL       → без параметров = DECIMAL(18,0)

ДОПУСТИМЫЕ ЗНАЧЕНИЯ ДЛЯ DECIMAL(5,2)
════════════════════════════════════
✅ 123.45     → 5 цифр (3 до, 2 после запятой)
✅ 99.99      → допустимо
✅ 0.01       → допустимо
✅ -500.00    → допустимо
✅ 999.99     → максимум
❌ 1234.56    → 6 цифр всего — ОШИБКА!
❌ 123.456    → 3 цифры после запятой — округлится или ошибка

ВАЖНОЕ ОТЛИЧИЕ ОТ FLOAT/REAL
═══════════════════════════════════════════════════════════════
DECIMAL                 │  FLOAT/REAL
════════════════════════╪═══════════════════════════════════════
Точный (exact)         │  Приблизительный (approximate)
Хранит числа точно     │  Может хранить с микропогрешностью
Идеален для денег      │  Плох для денег
Медленнее в вычислениях│  Быстрее
0.1 + 0.2 = 0.3 точно  │  0.1 + 0.2 может быть 0.30000000000000004

ПРИМЕР ОКРУГЛЕНИЯ
═══════════════════════════════════════════════════════════════
DECLARE @d DECIMAL(5,2) = 123.456  → станет 123.46 (округлится)

ПРИМЕРЫ ХРАНЕНИЯ ДАННЫХ
═══════════════════════════════════════════════════════════════
Деньги:    DECIMAL(19,4)  → достаточно для большинства валют
Целое:     DECIMAL(10,0)  → как INT, но больше диапазон
Проценты:  DECIMAL(5,2)   → например, 99.99%

ОШИБКА ПЕРЕПОЛНЕНИЯ
═══════════════════════════════════════════════════════════════
При попытке вставить число 1022018 в DECIMAL(5,2):
→ Arithmetic overflow error converting numeric to data type numeric

═══════════════════════════════════════════════════════════════
                    АНАЛИЗ ВАШЕЙ СИТУАЦИИ
═══════════════════════════════════════════════════════════════

В вашем коде:
@a13 [DECIMAL](7,0) = 1022018  → 7 цифр, 0 после запятой
В таблице: DECIMAL(5,2)        → всего 5 цифр, 2 после запятой

ПРОБЛЕМА: 1022018 — это 7 цифр, не влезает в 5 цифр.

СПОСОБЫ ИСПРАВЛЕНИЯ
═══════════════════════════════════════════════════════════════

Вариант 1 — изменить значение:
@a13 DECIMAL(5,2) = 1022.18  → 5 цифр: 1022.18

Вариант 2 — изменить тип колонки в таблице:
ALTER TABLE AKUZ.T_CDA_SECTION_IMM_VAC_CERT_ELEMENT 
ALTER COLUMN Dose DECIMAL(7,2)
