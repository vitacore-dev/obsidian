## Symptoms

- Запрос в MSSQL выполняется слишком долго или завис.
- Сессия потребляет ресурсы и мешает обработке других операций.
- Необходимо быстро завершить конкретный процесс (SPID).

## Scope

- Версия продукта: `all`
- Окружение: любое, где используется Microsoft SQL Server
- Инструменты: SQL-консоль, SSMS

## Diagnosis

1. Подключиться к SQL Server с правами на управление сессиями.
2. Получить список активных процессов:
   ```sql
   sp_whoisactive
   ```
3. Найти идентификатор нужного процесса (SPID).
4. Подтвердить, что выбран именно проблемный процесс.

## Resolution / Workaround

1. Завершить процесс по его идентификатору:
   ```sql
   KILL <spid>;
   ```
2. Проверить, что процесс больше не активен.
3. Если процесс возникает повторно, зафиксировать шаблон запроса и передать в разбор на инженерную команду.

## Escalation

Эскалировать, если:

- завершение процесса не дало эффекта;
- зависания повторяются системно;
- инцидент затрагивает SLA или критичные клиентские операции.

Эскалировать в:

- Команда: `DBA / Engineering On-call`
- Передать: SPID, SQL-текст (если доступен), время начала, пользователь, влияние на сервис
